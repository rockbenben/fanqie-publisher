# 番茄作家批量上传工具

> 把本地 Markdown / TXT 章节批量上传到[番茄作家](https://fanqienovel.com)，支持定时发布、立即发布、存草稿、修改内容与批量修改排期——全程图形界面，开箱即用。

![Python](https://img.shields.io/badge/Python-3.10%2B-3776AB?logo=python&logoColor=white)
![Playwright](https://img.shields.io/badge/Playwright-1.40%2B-2EAD33?logo=playwright&logoColor=white)
![Platform](https://img.shields.io/badge/Platform-Windows%20%7C%20macOS%20%7C%20Linux-lightgrey)
![License](https://img.shields.io/badge/License-MIT-green)

一次选好章节文件夹，工具会自动解析章节号、清洗 Markdown 格式、按你设定的时间表逐章发布，并在遇到平台限制时智能跳过、结束后给出可一键补传的失败清单。

## 最简上手

只想赶紧用？三步即可：

1. 安装 [Python 3.10+](https://www.python.org/downloads/)（Windows 安装时记得勾选 **Add Python to PATH**）
2. **Windows** 双击 `run.bat`；**macOS / Linux** 终端运行 `bash run.sh`——首次会自动装好依赖并启动
3. 点「登录/新建」，在弹出的浏览器里登录番茄账号，回到界面依次选作品 → 选章节文件夹 → 开始上传

> 更详细的图文步骤见 [使用指南](#使用指南)；卡住了看 [常见问题](#常见问题)。

## 功能特性

- **图形界面** — 全 GUI 操作，无需记命令
- **一键启动** — 首次运行自动安装依赖，开箱即用
- **新手引导** — 首次启动弹欢迎引导；顶部引导栏按「登录 → 选作品 → 选章节 → 上传」分步高亮并提示下一步，可「✕ 收起」，随时点「❓帮助」重看，各区块旁 `(?)` 有单独说明
- **多账号切换** — 支持多个作家账号，一键切换登录状态
- **批量上传** — 一次上传整个目录（含子文件夹）的 MD/TXT 文件，按文件名自然排序
- **五种操作模式** — 定时发布 / 立即发布 / 存草稿 / 修改内容 / 修改排期
- **智能排期** — 自动检测上次发布时间，多时间点均匀分配章节（如 `08:00,12:00,20:00`）；起始日期已过去时自动提醒
- **智能解析** — 从文件名或标题自动提取章节号（支持 第X章/回/节/话、数字前缀、chapter-N），自动去重同名章节
- **多卷支持** — 自动检测作品分卷，支持按卷筛选或「合并所有卷」一次性操作全部章节
- **章节号筛选** — 所有模式下均可按章节号过滤（`≤` / `≥` 第 N 章，或输入范围/组合如 `5-10`、`1,3,5-10`），只操作指定范围的章节
- **按修改日期筛选** — 按文件修改时间过滤，只操作指定时间前/后修改的章节
- **每日字数上限检测** — 触发平台当日字数上限时立即中止整批（继续提交只会重复撞限、徒增错误），把当前章与所有剩余未处理章节记入清单，结束时输出可直接补传的章节号
- **失败自动重试** — 单章失败自动重试（可配置次数），大批量操作更稳定
- **定时执行** — 设定未来某一时刻自动执行当前所选操作（单次）；启动前检查登录/作品/章节是否就绪，到点若有任务在跑会等待空闲，触发前自动刷新章节目录以使用最新文件
- **会话持久化** — 登录一次后保存状态，后续无需重复登录
- **配置记忆** — 操作模式、每天章数、发布时间、章节文件夹路径、选中作品、「合并所有卷」、定时执行时间等自动记忆
- **正文粘贴校验** — 自动检测正文是否写入成功，粘贴失败时立即报错并重试
- **Markdown 自动清理** — 上传时自动去除加粗、斜体、删除线、列表标记、HTML 标签、代码块等格式，保留纯文本
- **输入容错** — 发布时间支持单位数小时（`8:00`→`08:00`）、全角标点（`，` `：` `；`）自动转换
- **日志与排查** — 运行日志可一键导出；失败自动截图；警告/错误滚动写入 `fanqie_error.log`（2MB 自动轮转，保留 3 份备份）

## 快速开始

> 需要 [Python 3.10+](https://www.python.org/downloads/)，首次运行会自动安装其余依赖。
> Linux 用户还需安装 tkinter：`sudo apt install python3-tk`

| 平台 | 启动方式 |
| ---- | -------- |
| **Windows** | 双击 `run.bat` |
| **macOS / Linux** | 终端运行 `bash run.sh` |

首次运行会自动执行 `pip install -r requirements.txt` 和 `playwright install chromium`，之后直接启动 GUI。启动脚本每次都会幂等校验浏览器内核，缺失或版本不对会自动补装。

## 使用指南

> 首次打开会弹出**新手引导**总览；主界面顶部还有一条**引导栏**，按「登录 → 选作品 → 选章节 → 上传」实时高亮当前步骤并提示下一步。引导栏可点「✕ 收起」（记住选择），需要时点顶部「❓帮助」可重看引导，各区块旁的 `(?)` 有单独说明。

### 1. 登录与账号管理

点击「登录/新建」，输入一个**本地账号名称**（如 `作家A`，仅用于区分多个账号，不是番茄笔名），浏览器会自动打开登录页面，同时主窗口最小化以避免遮挡。屏幕右下角会出现一个醒目的浮动提示窗口：

- 登录完成后，点击绿色「登录完成，保存会话」按钮
- 如需取消，点击「取消」按钮或关闭提示窗口

登录期间会锁定相关操作（上传、刷新作品、切换账号、定时触发都会暂避），等登录完成或取消后再继续——避免两个浏览器同时读写登录状态、把会话错存到别的账号名下。

> ⚠️ 请先点「登录完成，保存会话」**再**关浏览器。若先把浏览器关了，会话无法读取，工具会明确报错并要求重新登录（不会把上一个账号的旧会话错存到新账号名下）。

**多账号切换：** 登录多个账号后，通过顶部下拉框直接切换。切换时自动刷新作品列表，每个账号独立记忆上次选中的作品。

### 2. 选择作品

登录成功后自动刷新作品列表。选择作品后会自动获取最新发布信息（章节名 + 时间），显示在操作模式区域。点击「章节管理 ↗」可直接在浏览器中打开章节管理页面。

### 3. 选择章节文件夹

默认使用项目目录下的 `chapters/` 文件夹，也可点击「浏览...」选择其他目录（路径自动记忆）。子文件夹中的 `.md`/`.txt` 文件也会被识别为有效章节，按文件夹名 → 文件名自然排序。「自动处理重名」默认勾选，避免番茄作家的同名章节限制。

**按修改日期筛选：** 勾选后展开筛选选项，可按文件修改时间过滤章节文件：

- 选择「早于」或「晚于」指定时间点
- 支持 `YYYY-MM-DD HH:MM`（精确到分钟）或 `YYYY-MM-DD`（当天 00:00）两种格式
- 格式错误时会显示红色提示并阻止加载，避免静默忽略
- 适用于只上传近期修改过的章节，或排除新改动的文件

### 4. 选择操作模式

| 模式 | 说明 |
| ---- | ---- |
| **定时发布** | 设置起始日期、每天章数和发布时间，自动排期发布 |
| **立即发布** | 上传后直接发布 |
| **存草稿** | 上传后保存为草稿，可在后台逐章检查再手动发布 |
| **修改内容** | 按章节号匹配已有章节，替换内容后保存，不修改发布时间设置 |
| **修改排期** | 批量修改平台待发布章节的排期日期/时间，不修改章节内容 |

定时发布、立即发布、修改内容模式下可勾选「稿件使用了AI创作」，对应番茄作家的「是否使用AI」申报选项。如果作品使用了 AI 工具辅助创作，请如实勾选。存草稿和修改排期模式不涉及此选项。

**章节号筛选：** 所有模式下均可勾选「按章节号筛选」，支持三种写法：

- 单个数字（如 `30`）— 配合 `≤` / `≥` 运算符做阈值筛选
- 范围（如 `5-10`，也支持 `5~10`）— 只操作该范围内的章节
- 逗号组合（如 `1,3,5-10`）— 单号与范围可混用，命中任意一项即保留（分隔符兼容中英文逗号、分号、顿号）

使用范围或组合格式时运算符下拉框自动置灰。未通过筛选的章节会自动跳过，预览底部以汇总行标注跳过数量（如 `[跳过 3 章: 3 筛选]`）。

#### 定时发布详细说明

- 起始日期和时间会根据上次发布信息自动建议（上次日期 +1 天）
- **多时间点：** 发布时间支持逗号分隔多个时间，如 `08:00,12:00,20:00`（也支持全角逗号 `，`、分号 `；`、单位数小时 `8:00`、全角冒号 `：`）
- **自动调整：** 时间点数量大于每天章数时，每天章数自动上调
- **均匀分配：** 每天章数大于时间点时，章节均匀分配到各时间段，同一时间段内按 +1 分钟递增（各时间段之间不会重叠）
- **顺序保证：** 同一天内各章的发布时刻严格按章节顺序递增；即使发布时间临近午夜（如 `23:58` 排 3 章），也会自动把这几章整体前移到放得下（`23:57/23:58/23:59`），不会出现后面的章节反而先发的乱序
- 预览区会实时显示每章的排期日期和时间

> **示例：** 每天 6 章、发布时间 `07:00,12:00,20:00` → `07:00, 07:01, 12:00, 12:01, 20:00, 20:01`

#### 修改内容模式

选择「修改内容」后，工具会自动从平台获取已有章节列表，按章节号与本地文件匹配。匹配成功的章节会替换内容并保存，不修改已有的发布时间设置。

- 多卷作品会自动检测并显示分卷选择器，可按卷筛选章节；勾选「合并所有卷」可一次性匹配全部卷的章节，无需逐卷切换
- 「合并所有卷」设置按作品记忆，切换作品时自动恢复
- 标题在章节间搬移（如本地重新编号）造成的「重复标题」拒绝会自动留待**批末二次尝试**——此时占用旧标题的章节已更新、冲突自然解除，无需干预
- **本地章节号重复**（如多卷子文件夹各自从 1 编号、或目录里残留旧副本）时只取首个文件，其余跳过并在日志中列出重复号——避免同一平台章节被后处理的文件覆盖。多卷各自编号的作品请按卷分别操作，不要勾选「合并所有卷」
- 平台上数字开头的特殊章节（如「2024新春番外」）不会被误认为第 2024 章参与匹配，会列在「未找到」跳过

#### 修改排期模式

选择「修改排期」后，工具自动获取平台章节列表，筛选出审核状态为「待发布」的章节，按章节顺序重新排期。不需要本地文件，直接在章节管理页面逐章修改发布日期和时间。

- 仅「待发布」状态的章节会被列出和修改
- 数字开头的特殊章节（如「2024新春番外」）没有可靠章节号，使用「按章节号筛选」时会被自动排除，不会被错误排进主线序列
- 多卷作品会自动检测并显示分卷选择器，可按卷筛选章节；勾选「合并所有卷」可批量操作全部卷
- 排期参数（起始日期、每天章数、发布时间）与定时发布模式共用
- 预览区实时显示每章将被修改为的新日期和时间
- 如果存在同名章节，操作前会弹窗提示，建议先在平台修改标题以确保排期准确

### 5. 预览与上传

章节预览区实时显示章节列表、字数统计和排期（启用筛选时字数只统计筛选后的章节）。确认无误后点击「开始上传」（修改模式下为「开始修改」），确认对话框会显示实际操作的章节数量（筛选启用时标注「已筛选」）。运行日志区实时显示进度，上传期间所有操作控件自动锁定，点击「停止」可随时中断。

> 中断请用「停止」按钮。如果直接关掉浏览器窗口，任务也会停下并照常给出「成功/失败」汇总；此时尚未处理的章节会一并记入清单（含可直接补传的章节号），不会被静默漏掉，只是已在途的那一章可能处于中间状态。若连续多章原因不明失败，工具会判定流程异常、提前中止整批，同样把剩余章节列入清单备查。

上传过程中如果平台提示当日字数上限（如「已到达当日发布字数上限」「提交字数超出每日上限」），工具会立即中止整批——继续提交后续章节多半重复撞限、还会触发别的拦截，徒增额外错误。当前章与所有剩余未处理章节都会记入清单，结束时统一列出（含可直接补传的章节号），次日接着发即可。每章正文粘贴后会自动校验字数，粘贴失败时立即报错并触发重试。

### 6. 定时执行（可选）

在「定时执行」区域填入执行时间（`YYYY-MM-DD HH:MM`），点击「启动定时」，工具会在到点时自动执行当前所选的操作模式（仅触发一次）。适合排期到流量更高的时段、或无人值守时自动上传。

- 启动前会检查登录状态、是否选好作品、是否有可操作章节，发现疏漏会提前提示
- 到点时若已有任务在运行（或正在登录），会等其空闲后再执行，不会丢弃本次定时
- 触发前约 60 秒自动刷新一次章节目录，确保使用等待期间的最新文件
- 执行时间会被记住，下次打开自动填入（已过期则建议为「当前时间 +1 小时」）
- 定时期间请保持程序运行；点击「取消定时」可随时停止

## MD 文件格式

每个 MD 或 TXT 文件对应一个章节：

```markdown
# 第 39 章 安定期

一周后。

周六下午两点。

林默和南俪并肩走在公园里。
```

- **首行**的 `# 标题` 用于提取章节号和标题
- 如果首行不是 `# 标题`，则从文件名提取；正文中部出现的 `# ` 行按正文处理（只去掉 `#` 记号、保留文字），不会影响标题或截断内容
- 正文中的 Markdown 格式标记会自动去除（加粗、斜体、删除线、标题、引用、列表、代码块、HTML 标签等）。HTML 清理只移除真实标签形状（如 `<div>`、`</p>`），正文里的颜文字 `(>_<)`、比较符号 `体温>38` 等不会被误删

**支持的文件名/标题格式：**

| 格式 | 示例 | 章节号 | 标题 |
| ---- | ---- | :----: | ---- |
| 数字前缀 | `001_新的旅程.md` | 1 | 新的旅程 |
| 数字+冒号 | `001：新的旅程.md` | 1 | 新的旅程 |
| 第X章（数字） | `第27章_重新开始.md` | 27 | 重新开始 |
| 第X章（中文） | `第十六章 发布会.txt` | 16 | 发布会 |
| 第X回 | `第二十七回 黛玉葬花.md` | 27 | 黛玉葬花 |
| 第X话/节 | `第16话 出发.md` | 16 | 出发 |
| chapter-N | `chapter-039.md` | 39 | _(从文件内 `#` 标题提取)_ |
| 纯文字 | `安定期.md` | _(无)_ | 安定期 |

## 配置文件

`config.json` 可调整默认行为（首次运行时自动生成，GUI 会自动保存部分设置）：

```json
{
  "delay_between_chapters": 3,
  "headless": false,
  "max_retries": 2,
  "default_mode": "schedule",
  "default_per_day": 2,
  "default_time": "08:00",
  "browser_timeout": 15000,
  "chapters_dir": "/path/to/your/chapters"
}
```

| 字段 | 说明 | 默认值 |
| ---- | ---- | ------ |
| `delay_between_chapters` | 章节间等待秒数 | `3` |
| `headless` | 隐藏浏览器窗口运行（后台静默上传） | `false` |
| `max_retries` | 单章上传失败最大重试次数 | `2` |
| `default_mode` | 默认操作模式（`schedule` / `publish` / `draft` / `edit` / `reschedule`） | `schedule` |
| `default_per_day` | 默认每天章数 | `2` |
| `default_time` | 默认发布时间，多个用逗号分隔（支持 `8:00` 等简写） | `08:00` |
| `browser_timeout` | 浏览器操作超时（毫秒，最小 1000；更小的值视为误填秒数并回退默认） | `15000` |
| `chapters_dir` | 章节文件夹路径（GUI 自动保存） | `chapters/` |

## 命令行用法

> 🛠️ **进阶 / 开发者向** —— 只用图形界面的话可跳过本节。

GUI 之外也支持纯命令行操作（`reschedule` 修改排期仅 GUI 支持）：

```bash
# 登录并保存会话
python fanqie_upload.py login

# 列出你的作品及 Book ID
python fanqie_upload.py books

# 上传（默认仅存草稿）
python fanqie_upload.py upload ./chapters --book-id <ID>

# 定时发布：从 3-14 起每天 2 章，三个时间点
python fanqie_upload.py upload ./chapters --book-id <ID> \
    --schedule 2026-03-14 --time 08:00,12:00,20:00 --per-day 2

# 立即发布
python fanqie_upload.py upload ./chapters --book-id <ID> --publish

# 修改已有章节内容（按章节号匹配）
python fanqie_upload.py upload ./chapters --book-id <ID> --edit
```

`upload` 子命令的常用参数：

| 参数 | 说明 |
| ---- | ---- |
| `--book-id` | 目标作品 ID（必填） |
| `--publish` | 直接发布（默认仅存草稿） |
| `--schedule DATE` | 定时发布起始日期 `YYYY-MM-DD` |
| `--time` | 发布时间，多个逗号分隔，如 `08:00,12:00,20:00` |
| `--per-day N` | 每天发布章数（默认 `1`） |
| `--edit` | 修改已有章节（按章节号匹配，不可与 `--publish`/`--schedule` 同用） |
| `--use-ai` | 申报「使用 AI 创作」 |
| `--unique-titles` | 自动给重复标题追加章节号后缀 |
| `--headless` | 无头模式（不显示浏览器） |
| `--delay N` | 章节间等待秒数（默认 `3`） |

完整参数见 `python fanqie_upload.py -h` 与 `python fanqie_upload.py upload -h`。

## 常见问题

**点「登录/新建」、填了名称后没有任何反应？**
最常见的原因是**浏览器内核（chromium）没装上**——`playwright` 这个 Python 包和它驱动的 chromium 浏览器是两个独立部分，只装了包、没装浏览器时，点登录会在打开浏览器那一步失败。新版已针对此做了处理：

- `run.bat` / `run.sh` 启动时都会幂等地执行 `playwright install chromium`，缺失或版本不对会自动补装（升级 Playwright 后浏览器没同步也能修好）
- 万一仍缺，点登录后会**弹窗提示**「缺少浏览器内核」，可一键自动下载，或按提示手动运行 `python -m playwright install chromium`
- 顺带一提：登录框里填的是**本地账号名称**（用于区分多个账号），不是番茄笔名；真正的登录/创建作家是在弹出的浏览器里手动完成的

如果你用的是较旧版本且遇到此问题，升级到最新版或重新运行 `run.bat` / `run.sh` 即可。

**登录状态过期了怎么办？**
点击「登录/新建」，输入对应的账号名称重新登录即可，会提示是否覆盖旧的登录状态。若登录状态文件损坏（如进程被强制中断留下半截文件），工具会自动忽略并提示重新登录，不会启动失败。

**如何切换多个作家账号？**
每个账号登录一次后即可通过顶部下拉框切换。登录状态分别保存在 `.auth_账号名.json` 文件中。

**提示「已到达当日发布字数上限」？**
这是番茄作家平台的每日限制，工具检测到后会立即中止整批（继续提交只会重复撞限、徒增错误），并把当前章与所有剩余未处理章节一并列入结束汇总。汇总末尾会直接给出压缩好的章节号（如 `79-114`）——次日把它原样粘贴到「按章节号筛选」输入框，即可只补传这些章节。

**提示「本书中存在重复标题，请修改后再发布」？**
分两种情况：

1. **标题搬移的临时冲突**（常见）——本地重新编号把标题从一章挪到了另一章，批量按章节号顺序处理时，新章会先于旧章提交同名标题而被拒。工具会自动把这类章节留待批末二次尝试（此时旧章已更新、标题已释放），通常无需干预。
2. **真正撞名**——与平台上未参与本次批量的章节同名，二次尝试也会失败并留在失败清单。需在平台手动改掉其中一个标题，再用「按章节号筛选」补传。

**修改内容时平台弹「请在发布时间前30分钟提交修改内容，否则无法完成修改」？**
这是平台规则提示，工具会自动点掉「我知道了」继续执行，不会卡住。但提示本身的规则绕不过：**距定时发布不足 30 分钟时提交的修改可能不生效**——即使日志显示「已保存修改」，也建议对临近发布时间的章节抽查确认。

**上传失败怎么排查？**
按这个顺序看：

1. **结束汇总** — 批量结束时日志会统一列出「失败章节及原因」清单
2. **平台提示原文** — 失败瞬间捕获到的平台弹窗文字会以「页面提示: …」记入日志；超时类失败还会附带当时的页面 URL 和确认按钮状态
3. **错误截图** — 失败章节自动截图保存在项目目录下（如 `error_0_chapter-039.png`）
4. **完整日志** — `fanqie_error.log` 记录了全过程，可回溯操作

如果日志显示「页面提示」里出现了工具不认识的失败文案，欢迎反馈以便补充识别词库。

**任务跑完了，界面却像卡住一样、迟迟不出汇总？**
个别情况下浏览器子进程会卡在退出阶段（实测曾长达 41 分钟），而关闭浏览器要等它整体退出才返回。章节其实早已提交成功，只是收尾被拖住。工具已对关闭浏览器、保存登录状态等收尾步骤加了超时保护：关不掉就放弃等待，完成汇总和「成功/失败」结果会立即给出，不再被挂死的浏览器进程拖住。日志中出现「关闭浏览器超过 N 秒未完成（浏览器进程疑似挂死）」属于此情况，不影响本次结果。

**可以断点续传吗？**
目前不支持自动断点续传，但很接近：批量结束汇总会把所有未成功/未处理的章节压成一段章节号（如 `79-114`），直接复制粘贴到「按章节号筛选」即可只补传这些章节。也可以手动设置 `≥ 第 N 章` 跳过已上传的章节，或直接输入未传章节的组合（如 `5,8,10-20`），配合调整起始日期继续上传。

**还能用命令行吗？**
可以，见 [命令行用法](#命令行用法)。运行 `python fanqie_upload.py -h` 查看完整参数。

**如何修改已发布的章节？**
GUI 选择「修改内容」模式，工具会按章节号自动匹配；命令行使用 `python fanqie_upload.py upload ./chapters --book-id <ID> --edit`。

**如何批量修改排期？**
GUI 选择「修改排期」模式，工具自动获取待发布章节并重新排期。仅审核状态为「待发布」的章节可修改排期（此功能仅 GUI 支持）。
