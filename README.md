# 番茄作家 MD 批量上传工具

将本地 Markdown 文件批量上传到 [番茄作家](https://fanqienovel.com) 平台作为小说章节，支持存草稿、立即发布、定时发布和修改已有章节。

## 功能特性

- **图形界面** — 全 GUI 操作，无需记命令
- **一键启动** — 首次运行自动安装依赖，开箱即用
- **多账号切换** — 支持多个作家账号，一键切换登录状态
- **批量上传** — 一次上传整个目录的 MD/TXT 文件，按文件名自然排序
- **四种操作模式** — 存草稿 / 立即发布 / 定时发布 / 修改已有章节
- **智能排期** — 自动检测上次发布时间，支持多时间点发布（如 08:00,12:00,20:00）
- **智能解析** — 从文件名或标题自动提取章节号，自动去重同名章节
- **修改日期筛选** — 按文件修改时间过滤，只上传指定时间前/后修改的章节
- **每日字数上限检测** — 达到平台当日发布字数上限时自动停止，避免报错
- **失败自动重试** — 单章失败自动重试（可配置次数），大批量操作更稳定
- **会话持久化** — 登录一次后保存状态，后续无需重复登录
- **配置记忆** — 发布模式、每天章数、发布时间、选中作品等自动记忆
- **正文粘贴校验** — 自动检测正文是否写入成功，粘贴失败时立即报错并重试
- **日志导出** — 运行日志可一键导出为文本文件，方便保存和排查
- **错误截图** — 上传失败时自动保存截图，方便排查

## 快速开始

> 需要 [Python 3.10+](https://www.python.org/downloads/)，首次运行会自动安装其他依赖。
> Linux 用户还需安装 tkinter：`sudo apt install python3-tk`

- **Windows** — 双击 `run.bat`
- **macOS / Linux** — 终端运行 `bash run.sh`

首次运行会自动执行 `pip install` 和 `playwright install chromium`，之后直接启动 GUI。

## 使用流程

### 1. 登录与账号管理

点击「登录/新建」，输入账号名称（如"笔名A"），浏览器会自动打开登录页面，同时主窗口最小化以避免遮挡。屏幕右下角会出现一个醒目的浮动提示窗口：

- 登录完成后，点击绿色「登录完成，保存会话」按钮
- 如需取消，点击「取消」按钮或关闭提示窗口

登录期间按钮自动锁定，防止重复操作。

**多账号切换：** 登录多个账号后，通过顶部下拉框直接切换。切换时自动刷新作品列表，每个账号独立记忆上次选中的作品。

### 2. 选择作品

登录成功后自动刷新作品列表。选择作品后会自动获取最新定时发布信息（章节名 + 时间），显示在发布模式区域。

点击「章节管理 ↗」可直接在浏览器中打开章节管理页面。

### 3. 选择章节目录

默认使用项目目录下的 `chapters/` 文件夹，也可点击「浏览...」选择其他目录。

「自动去重标题」默认勾选，避免番茄作家的同名章节限制。

**筛选修改日期：** 勾选后展开筛选选项，可按文件修改时间过滤章节文件：
- 选择「早于」或「晚于」指定时间点
- 支持 `2025-01-01 08:00`（精确到分钟）或 `2025-01-01`（当天 00:00）两种格式
- 适用于只上传近期修改过的章节，或排除新改动的文件

### 4. 选择发布模式

| 模式         | 说明                                                       |
| ------------ | ---------------------------------------------------------- |
| **存草稿**   | 上传后保存为草稿，可在后台逐章检查再手动发布               |
| **立即发布** | 上传后直接发布                                             |
| **定时发布** | 设置起始日期、每天章数和发布时间，自动排期发布             |
| **修改**     | 按章节号匹配已有章节，替换内容后保存，不修改发布时间设置   |

发布模式下可勾选「稿件使用了AI创作」，对应番茄作家的"是否使用AI"申报选项。如果作品使用了AI工具辅助创作，请如实勾选。

#### 定时发布详细说明

- 起始日期和时间会根据上次发布信息自动建议（上次日期 +1 天）
- **多时间点：** 发布时间支持逗号分隔多个时间，如 `08:00,12:00,20:00`
- **自动调整：** 时间点数量大于每天章数时，每天章数自动上调
- **智能补齐：** 每天章数大于时间点时，自动按 +1 分钟间隔补齐时间
- 预览区会实时显示每章的排期日期和时间

#### 修改模式

选择「修改」后，工具会自动从平台获取已有章节列表，按章节号与本地文件匹配。匹配成功的章节会替换内容并保存，不修改已有的发布时间设置。

### 5. 预览与上传

章节预览区实时显示章节列表、字数统计和排期。确认无误后点击「开始上传」，运行日志区实时显示进度。上传期间所有操作控件自动锁定，点击「停止上传」可随时中断。

上传过程中如果平台提示「已到达当日发布字数上限」，工具会自动检测并停止上传。每章正文粘贴后会自动校验字数，粘贴失败时立即报错并触发重试。

运行日志可点击「导出日志」保存为文本文件。

## MD 文件格式

每个 MD 或 TXT 文件对应一个章节：

```markdown
# 第 39 章 安定期

一周后。

周六下午两点。

林默和南俪并肩走在公园里。
```

- 第一个 `# 标题` 用于提取章节号和标题
- 如果没有 `# 标题`，则从文件名提取
- 正文中的 Markdown 格式标记会自动去除

**支持的文件名/标题格式：**

| 格式         | 示例                   | 章节号 | 标题                    |
| ------------ | ---------------------- | ------ | ----------------------- |
| 数字前缀     | `001_新的旅程.md`      | 001    | 新的旅程                |
| 第X章(数字)  | `第27章_重新开始.md`   | 27     | 重新开始                |
| 第X章(中文)  | `第十六章 发布会.txt`  | 16     | 发布会                  |
| chapter-N    | `chapter-039.md`       | 039    | _(从文件内 # 标题提取)_ |
| 纯文字       | `安定期.md`            | _(无)_ | 安定期                  |

## 配置文件

`config.json` 可调整默认行为（GUI 会自动保存部分设置）：

```json
{
  "delay_between_chapters": 3,
  "headless": false,
  "max_retries": 2,
  "default_mode": "schedule",
  "default_per_day": 2,
  "default_time": "08:00",
  "browser_timeout": 15000
}
```

| 字段                     | 说明                                   | 默认值     |
| ------------------------ | -------------------------------------- | ---------- |
| `delay_between_chapters` | 章节间等待秒数                         | `3`        |
| `headless`               | 隐藏浏览器窗口运行（后台静默上传）     | `false`    |
| `max_retries`            | 单章上传失败最大重试次数               | `2`        |
| `default_mode`           | 默认发布模式 (draft/publish/schedule/edit) | `schedule` |
| `default_per_day`        | 默认每天章数                           | `2`        |
| `default_time`           | 默认发布时间，多个用逗号分隔           | `08:00`    |
| `browser_timeout`        | 浏览器操作超时（毫秒）                 | `15000`    |

```

```

## 常见问题

**登录状态过期了怎么办？**
点击「登录/新建」，输入对应的账号名称重新登录即可，会提示是否覆盖旧的登录状态。

**如何切换多个作家账号？**
每个账号登录一次后即可通过顶部下拉框切换。登录状态分别保存在 `.auth_账号名.json` 文件中。

**提示「已到达当日发布字数上限」？**
这是番茄作家平台的每日限制，工具检测到后会自动停止。次日再继续上传即可。

**上传失败怎么排查？**
失败的章节会自动截图保存在项目目录下（如 `error_0_chapter-039.png`），查看截图了解页面状态。

**可以断点续传吗？**
目前不支持。将已上传的 MD 文件移出目录，调整起始日期后重新上传。

**还能用命令行吗？**
可以。运行 `python fanqie_upload.py -h` 查看完整命令行参数。

**如何修改已发布的章节？**
选择「修改」模式，工具会按章节号自动匹配。命令行使用 `python fanqie_upload.py upload ./chapters --book-id ID --edit`。
